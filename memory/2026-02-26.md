## Layer 5 Phase 2 ‚Äî Multi-Agent Orchestration + I/O Channels (later)

All Priority 2 and 3 items completed and live-tested with `google/gemma-3-4b`.

### Tasks Completed

**Task 2.1: Sub-agent Spawning** ‚Äî `clawmacs/subagents`
- `spawn-subagent` launches `run-agent` in a `bt:make-thread`
- Returns `subagent-handle` with thread, session, status, result, error slots
- `subagent-wait` blocks (with optional timeout) using lock + condvar
- `subagent-status` polls (non-blocking)
- `subagent-kill` destroys thread, sets `:killed`
- Live tested: spawned subagent, waited, got `:done` result

**Task 2.2: Agent Registry** ‚Äî `clawmacs/registry`
- `*agent-registry*` hash table, thread-safe with bt lock
- `register-agent`, `find-agent`, `list-agents`, `unregister-agent`, `clear-registry`
- `define-agent` macro for declarative spec creation
- `agent-spec` struct: name, role, model, system-prompt, tools, client
- `instantiate-agent-spec` ‚Üí `make-agent`

**Task 3.1: Channel Protocol** ‚Äî `clawmacs/channels`
- Abstract `channel` CLOS class + generic functions: `channel-send`, `channel-receive`, `channel-poll`, `channel-close`, `channel-open-p`
- `repl-channel` ‚Äî wraps `*standard-input*`/`*standard-output*` streams
- `queue-channel` ‚Äî thread-safe FIFO queue (lock + condvar), supports timeout
- Conditions: `channel-closed-error`, `channel-timeout-error`

**Task 3.2: HTTP API Server** ‚Äî `clawmacs/http-server`
- Hunchentoot-based, port 7474 default
- `POST /chat` ‚Äî synchronous JSON, runs `run-agent`, returns `{"response": ..., "session_id": ...}`
- `POST /chat/stream` ‚Äî SSE via `*on-stream-delta*` hook
- `GET /agents` ‚Äî lists registry entries
- `GET /sessions` ‚Äî lists active HTTP sessions
- `start-server`, `stop-server`, `server-running-p`, `restart-server`
- Live tested all endpoints
